# 子弹/技能/角色/僵尸 差异化音效生成 Prompt

## 背景与目标

现在游戏里不同种类的子弹命中/开火确实已经有区分——`core/audio/audio_manager.gd` 里 8 把武器各自独立的开火声(`shot_autocannon`/`shot_flamethrower`/...)、5 种元素各自独立的命中声(`hit_physical`/`hit_fire`/`hit_ice`/`hit_lightning`/`hit_poison`)都是完全不同的音频文件，不是重复/占位。**这部分不用动。**

真正缺失、这次要补的是三块完全没有专属音效、目前要么复用通用音效、要么干脆没有声音的地方：

1. **16 个技能各自的"生效瞬间"音效**——现在技能是纯数值加成(分裂/穿透/暴击率等)，触发时没有任何专属反馈声，全部只能听见共用的命中/开火声。
2. **4 个角色的专属语音/技能音效**——4 个角色的开场识别音完全没有；4 个专属主动技能里，只有"熔毁爆发"(muzzle_fire)/"冰川领域"(muzzle_ice)/"雷暴领域"(muzzle_lightning)勉强复用了对应元素的开枪音效，"弹幕齐射"(钢铁先锋的主动技)现在用的是 `level_up`(角色升级提示音)——完全不搭，这是本次最需要修正的一处。
3. **20 种普通僵尸机制各自的攻击/技能音效**——现在所有僵尸共用同一个 `enemy_death` 死亡声，尖啸/冲锋/召唤/分裂/变异/狂暴……这些完全不同的机制在听感上毫无区别。

## 通用技术规格(全部音效统一遵守)

- **格式**：WAV，44.1kHz，单声道(参考现有 `sfx_hit_fire.wav` 等文件的格式，保持一致，方便引擎按同一套逻辑加载)。
- **时长参考**：一次性打击类音效 0.1–0.3 秒；持续型(蓄力/光环/领域)可到 2–6 秒并做好首尾可裁切/循环处理；角色开场语音类 0.4–0.8 秒。
- **响度**：峰值不要顶到 0dB，留出混音余量(参考现有素材峰值大约在 -3dB 到 -6dB 区间)，避免新音效比旧音效明显更响或更轻。
- **风格基调**：整体是废土科幻丧尸射击游戏，音色偏工业/机械/合成器电子风，而不是奇幻魔法风——尤其技能音效应该听起来像"弹药/枪械/护甲机构"在做反应，僵尸音效应该是"血肉+呻吟+偶尔机械义体"的混合质感，不要做成纯魔法特效音。
- **元素识别音色锚点**(和技能卡/图标的配色标准对应，保证听觉和视觉认知一致)：
  - 火(fire)：噼啪燃烧、气流呼啸、偏暖的失真
  - 冰(ice)：清脆结晶碎裂、低温空气收缩的"嘎吱"声
  - 雷/闪电(lightning)：短促尖锐的电流"滋滋/劈啪"声，游戏内闪电的标准视觉色是**金黄色**而不是蓝色，音色上也建议偏"高压电弧"而不是"科幻镭射"
  - 毒(poison)：粘稠气泡、腐蚀嘶嘶声

## A. 技能生效音效(16个，新增 key 前缀 `skill_`)

每条给出：技能名 / 触发时机 / 音效描述 / 建议文件名。

1. **分裂弹**(命中后子弹分裂成2发)——命中瞬间播放。清脆的"啪"分裂声，紧跟一个极短的高频回声，像是弹体在空中裂成两半。约0.15s。→ `sfx_skill_split_shot.wav`
2. **穿透弹**(穿透1个额外敌人)——穿透瞬间。低沉的"噗嗤"闷响穿刺感，比普通命中声更扎实、略带延音。约0.15s。→ `sfx_skill_pierce.wav`
3. **多重射击**(额外弹丸)——开火瞬间，与武器开火声叠加播放的一层。轻微的"多重回声"叠层感，让人听出不止一发子弹同时打出。约0.1s。→ `sfx_skill_multishot.wav`
4. **减速力场**(范围减速)——力场生效瞬间。低频"嗡"一声脉冲展开，带一点点降调处理(pitch-bend down)营造"时间变慢"的听感。约0.3s。→ `sfx_skill_slow_field.wav`
5. **追踪弹**(子弹自动追踪)——转向锁定瞬间。柔和的电子"滴"锁定提示音，类似雷达/导弹锁定音效但更轻巧。约0.1s。→ `sfx_skill_homing.wav`
6. **弱点暴击**(暴击伤害)——暴击命中瞬间。厚重的金属"哐"重击声，带一点高频拖尾，突出"处决感"，比普通命中声更有分量。约0.2s。→ `sfx_skill_critical.wav`
7. **防线屏障**(获得护盾)——护盾生成瞬间。由低到高上扬的电子"充能起罩"声。约0.25s。→ `sfx_skill_barrier.wav`
8. **淘金弹链**(击杀掉落金币增加)——击杀触发瞬间(叠加在现有金币音效之上的变体)。比现有 `sfx_gold_pickup.wav` 更丰富清脆的"哗啦"金币群声。约0.2s。→ `sfx_skill_gold_rush.wav`
9. **跳弹连锁**(子弹弹射连锁)——每次弹射命中一个节点时。短促的"叮"弹跳声，音高可随连锁次数逐次略微升高。约0.08s/次。→ `sfx_skill_ricochet.wav`
10. **急速装填**(射速提升)——效果生效瞬间。机械"哒哒哒"快速上膛/齿轮加速声。约0.2s。→ `sfx_skill_salvo.wav`
11. **燃烧弹**(命中点燃持续伤害)——命中点燃瞬间。"刺啦"点火声+细小噼啪燃烧声，要能和武器自带的 `hit_fire` 区分——这个更强调"点燃"瞬间而不是爆炸。约0.2s。→ `sfx_skill_incendiary.wav`
12. **冰霜弹**(命中减速+冰伤)——命中冻结瞬间。清脆的"咔嚓"结冰声。约0.15s。→ `sfx_skill_cryo.wav`
13. **特斯拉弹**(命中连锁放电)——命中连锁瞬间。"滋滋"跳电声，要比武器自带的 `hit_lightning` 更有"从一个目标跳到下一个目标"的连续跳跃感（可以做成2-3个快速连续的电流脉冲）。约0.2s。→ `sfx_skill_tesla.wav`
14. **毒蚀弹**(命中持续中毒)——命中瞬间。"咕嘟"腐蚀气泡声。约0.15s。→ `sfx_skill_venom.wav`
15. **蓄能重击**(攻击附带蓄力伤害)——分两段：蓄力过程中持续的低频"嗡鸣"充能声(可循环，约0.5-1s一个循环)；蓄满释放瞬间一声扎实的"砰"重击声(约0.15s)。→ `sfx_skill_charge_shot_charge.wav` + `sfx_skill_charge_shot_release.wav`
16. **战术回收**(额外选卡刷新机会)——触发瞬间。清脆的卡牌"唰"洗牌/循环声，要和现有通用的 `sfx_reroll.wav` 有区分度，更强调"回收/再生"的机械循环感而不是随机抽卡感。约0.2s。→ `sfx_skill_recycle.wav`

## B. 角色专属音效(4个角色，新增 key 前缀 `char_`/`sig_`)

每个角色给出：开场识别音(战斗开始时短促报出角色个性的一声，不需要真人语音台词，用音效/短促人声气口即可) + 专属主动技能释放音效。

### 钢铁先锋(vanguard，元素：物理，定位：均衡)
- 开场识别音：短促的金属装弹"咔嗒"上膛声 + 低沉的一声硬朗呼气，体现"老兵/前锋"的沉稳感。约0.5s。→ `sfx_char_vanguard_intro.wav`
- 专属主动技"**弹幕齐射**"(短时间提高攻速+额外弹道穿透，持续6秒的连续驳火)：**这是当前最需要修的一处**——现在这个技能释放时用的是角色升级音效(`level_up`)，完全文不对题。需要一段连续 5 发左右、节奏由慢到快的重型机炮连射轰鸣声，带金属回声，营造"火力压制"听感，总长匹配技能持续时间，首尾可裁切循环。约4-6s。→ `sfx_sig_vanguard_railvolley.wav`

### 火焰少年(blaze，元素：火，定位：爆发)
- 开场识别音：短促的热血少年呐喊气口 + 一点火焰"呼"起的声音。约0.5s。→ `sfx_char_blaze_intro.wav`
- 专属主动技"**熔毁爆发**"(锁定高威胁目标引爆火焰范围伤害，4段脉冲爆发)：4次脉冲式"轰"爆燃声，每次伴随烈焰扩散的"呼"声，低频提供厚重爆炸感、高频提供火星噼啪细节。约3-4s(4段脉冲连续)。→ `sfx_sig_blaze_meltdown.wav`(现有实现里这个技能复用的是 `muzzle_fire`，建议换成这条专属的)

### 冰霜少女(frost，元素：冰，定位：控制)
- 开场识别音：冷静少女的短促冷吟气口 + 一声清脆的冰晶碎裂点缀。约0.5s。→ `sfx_char_frost_intro.wav`
- 专属主动技"**冰川领域**"(防线前展开寒冰领域，持续5秒周期冰伤+减速，4波)：持续的低温"嘎吱"冰层扩张声打底 + 空气凝结呼啸声，4个波次对应4次轻微增强的冰裂声层叠。约5s(匹配技能持续时间)。→ `sfx_sig_frost_glacier.wav`(现有实现里这个技能复用的是 `muzzle_ice`，建议换成这条专属的)

### 电弧少女(volt，元素：闪电，定位：高速)
- 开场识别音：活泼少女的俏皮气口 + 一声电流"滋"的点缀。约0.5s。→ `sfx_char_volt_intro.wav`
- 专属主动技"**雷暴领域**"(向多名高威胁敌人释放雷击+震击)：一道主雷击"劈啪"巨响开场，紧接多道分支闪电噼啪连锁声呼应多个目标被同时命中，注意闪电音色统一用金黄高压电弧质感而不是蓝色镭射感。约1.5-2s。→ `sfx_sig_volt_storm.wav`(现有实现里这个技能复用的是 `muzzle_lightning`，建议换成这条专属的)

## 输出与接入清单

全部音效放到 `assets/production/audio/sfx/` 目录下，按上面表格里的文件名。生成完之后：

1. 在 `core/audio/audio_manager.gd` 的 `SFX` 字典里按对应 key 前缀补上新条目(key 名去掉 `sfx_` 前缀和 `.wav` 后缀，例如文件 `sfx_skill_pierce.wav` 对应 key `"skill_pierce"`)。
2. 具体在哪个函数里触发播放，由 Codex 结合 `gameplay/battle/battle.gd` 里技能效果实际生效的位置(比如 `_apply_damage_reduction_aura`、各 `_process_xxx_skill` 系列函数、`_on_projectile_hit_confirmed` 里技能效果分支等)自行判断接入点；角色专属技能音效替换点很明确——就是 `_cast_vanguard_railvolley`/`_cast_blaze_meltdown`/`_cast_frost_glacier`/`_cast_volt_storm` 这四个函数里调用 `_active_skill_cast_intro(...)` 时传入的 `sfx_id` 参数，从当前占位的 `"level_up"`/`"muzzle_fire"`/`"muzzle_ice"`/`"muzzle_lightning"` 换成新增的 `"sig_vanguard_railvolley"`/`"sig_blaze_meltdown"`/`"sig_frost_glacier"`/`"sig_volt_storm"`。

## 校验

全部接入后跑：

```
python3 tools/validate_data.py
python3 tools/check_release_candidate.py
godot --headless --path . --script tools/m1_smoke_test.gd
```

并建议做一次真机/编辑器内实际过一遍关卡，确认没有播放报错(缺文件会在 `AudioManager._load_sfx` 里静默 `load()` 失败返回 null，不会崩溃，但会导致该音效完全没声音，容易漏测)。
