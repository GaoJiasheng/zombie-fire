# 01 · 核心玩法

## 1. 屏幕与战场布局（竖屏 9:16）

逻辑分辨率 **1080 × 1920**。从上到下分区：

```
┌─────────────────────────────┐  y=0
│  顶部 HUD（关卡/波次/暂停）        │  0–140
├─────────────────────────────┤
│                             │
│        僵尸生成区（顶）          │  140–360  ← 僵尸从这里入场
│                             │
│      战 场（僵尸下压区）         │  360–1500 ← 主战斗区域
│                             │
│   ┌──── 基地防线 ────┐         │  1430–1500 ← 到达后停住并持续攻击基地
│   │   炮塔（玩家）       │       │  1620–1780
├─────────────────────────────┤
│  底部 HUD（技能槽×5 / 宝宝）      │  1780–1920
└─────────────────────────────┘  y=1920
```

- **炮塔**固定在底部中线，不能左右移动（塔防定位）。它**自动持续开火**。
- **僵尸**从顶部生成，沿屏幕向下推进，目标是在基地门前站住并持续攻击防线。
- 单一开阔战场（非 PvZ 多路分线），子弹海风格，强调"瞄哪、用什么技能"。

## 2. 三种玩家操作（iOS / macOS 同一套逻辑）

| 操作 | iOS（点击） | macOS（鼠标） | 作用 |
|---|---|---|---|
| **瞄准** | 手指按住战场拖动 | 鼠标移动/按住拖动 | 炮口跟随指向点；松开后保持最后朝向并自动锁定该方向最近目标 |
| **开火** | 自动 | 自动 | 永远自动，不需要玩家点 |
| **放技能** | 点底部技能图标 | 点技能图标 / 数字键 1-5 | 触发该主动技能（有冷却） |
| **切目标** | 双击某僵尸 | 右键某僵尸 | 手动锁定优先攻击该目标 |
| **切换目标策略** | 长按锁定按钮/设置 | HUD 小按钮/快捷键 | 最近 / 越线威胁 / 精英Boss / 血少，影响自动锁定 |

设计要点：
- **瞄准是核心手感**。炮口有转向速度（不是瞬移），玩家通过提前量、走位预判来打高价值目标（如死灵法师、自爆者）。
- 不强制持续操作：松手后炮塔自动锁"当前朝向扇区内最近/最高威胁"目标，照顾休闲手感。**主动操作 = 更高收益**（精准点杀关键怪），而非"必须一直操作否则团灭"。
- 技能分**主动**（玩家点，有 CD）与**被动**（携带即生效）。携带的 3~5 技能里既可能是主动也可能是被动，槽位通用。

### 目标选择规则（必须可信）

成熟同类游戏最容易被吐槽的是"该打的不打"。本作的默认自动瞄准必须像一个靠谱副驾驶：

1. **越线威胁优先**：距离基地防线最近、速度最快、`breach_damage` 高的敌人加权最高。
2. **机制怪优先**：自爆者、尖啸者、死灵法师、守卫、冲锋兵等带 `threat_tag` 的敌人优先。
3. **Boss/精英优先**：当 Boss 或精英进入有效输出窗口时，除非有即将越线的高危怪，否则优先集火。
4. **玩家锁定最高**：双击/右键锁定后，炮塔、追踪弹、宝宝技能都优先服务该目标，直到目标死亡/脱离/玩家取消。

UI 必须给锁定圈、威胁标记和"免疫/抗性/弱点"即时提示，避免玩家怀疑系统乱打（见 `10`/`13`）。

## 3. 自动开火规则

- 主炮有 `fire_rate`（发/秒）、`damage`、`element`、`projectile`（弹道形态，见 `06`）。
- 子弹沿炮口方向飞出；命中判定、穿透/分裂/弹射由"主炮基础形态 + 携带技能"叠加决定。
- 不消耗弹药、无装填（手感顺滑优先）。部分主炮有"过热/蓄能"机制制造节奏（见 `06`）。

## 4. 波次与关卡节奏

每关是一段**波次脚本**（`levels.json` 定义，见 `08`、`data/schema.md`）：

- 一关由 N 个 **wave** 组成（普通关 5~8 波，Boss 关含 1 个 boss wave）。
- 每波 = 一组 `{僵尸类型, 数量, 入场节奏, 入场位置}`。
- 波与波之间有短暂间隙（约 2~4 秒）让玩家喘息/放技能。
- **局内经验**随击杀积累，满一管触发一次技能选卡（见 `03` 的两层升级逻辑）。
- 技能选卡采用**可控随机**：系统按携带技能、角色倾向、关卡弱点、当前空槽和局面压力加权出 3 张；玩家有有限刷新/锁定/跳过，保证有惊喜但不被烂牌卡死。
- 关卡时长目标：普通关 **90~150 秒**，Boss 关 **150~240 秒**。

波次强度由"建议等级 + 难度系数"驱动（见 `08`/`09`），保证：
- 达到建议等级、装备正常强化的玩家，**靠合理操作能拿 2~3 星**。
- 低于建议等级也能打，但更吃操作、可能只拿 1 星。

## 5. 基地血量与胜负

- 基地有 `base_hp`（随玩家养成可提升，见 `02`/`03`）。
- 僵尸抵达基地防线 → 停在门前进入攻击状态；按自身攻击节奏持续扣基地血，直到被击杀。
- 不同僵尸的基地攻击由其 `mechanic` 与 `bd_coef` 派生：高速怪低伤高频，重甲怪高伤低频，腐蚀/自爆/支援型有不同 VFX 与攻击节奏。
- **失败**：基地血量归 0。可**无限重试**，已掉落金币保留（见 `02`）。
- **胜利**：清完全部波次 / 击杀 Boss，且基地血量 > 0。

## 6. 三星制（参考《消消乐》三星树）

通关时按**基地剩余血量百分比**结算星级：

| 剩余基地血量 | 星级 |
|---|---|
| 100%（满血通关） | ★★★ |
| 50% ~ 99% | ★★ |
| > 0% ~ 49% | ★ |
| 0%（失败） | 0（不计通关） |

- 星是**最珍贵的资源**，每关首次达成某星级时**累计入账**（见下）。
- 关卡选择界面是一条**关卡之路（树/线）**，每关显示已得星数，鼓励回刷满星。
- **补星**：回刷更高星级，只补差额（拿过 ★，后来打出 ★★★，补 2 星）。
- 三星追求不能变成焦虑：首通永远解锁下一关；三星更多用于额外星、外观和挑战目标，核心进度不要求关关满血。

### 星的产出与用途（与金币区分）

| 资源 | 来源 | 用途 | 稀有度 |
|---|---|---|---|
| **金币 gold** | 打怪掉落（关内） | 强化/购买装备 | 常见，刷得到 |
| **星 star** | 通关结算（按表现） | 解锁新主炮/新技能/新角色/皮肤 | 珍贵，关联进度与操作水平 |

> 设计意图：金币靠"肝"，星靠"通关质量"。想解锁高级内容必须**真的打好关卡**，而不是无脑刷。
> 全 99 关 × 最多 3 星 = **297 星上限**，star 解锁内容总价 < 297，确保正常通关玩家能逐步解锁全部内容（消耗曲线见 `09`）。

## 7. 单局流程图

```
进入关卡
  ├─ 出现 wave 1 …… wave N（按脚本）
  │     ├─ 玩家瞄准 + 自动开火 + 适时放主动技能
  │     ├─ 击杀 → 掉金币（自动/宝宝拾取）+ 攒局内经验
  │     └─ 局内经验满 → 三选一（填技能槽 / 升级在场技能）
  ├─ 僵尸抵达防线 → 驻留攻击基地
  └─ 全波清空 / Boss 死亡
        ├─ 基地血>0 → 胜利 → 按剩余血结算 1~3 星 + 金币入账
        └─ 基地血=0 → 失败 → 可重试（保留已得金币）
```

## 8. 反挫败设计

- **无体力**：随便打、随便重试。
- **不做一键扫荡/刷图**：重打必须真打（保留操作乐趣与价值）。
- **失败保金币**：每次进本掉的金币都进账，越打越接近"能升级"。
- **建议等级提示**：低于建议等级会标黄提示"有挑战"，但不禁止进入。
- **首通奖励**：每关首次通关额外给一笔金币 + 解锁下一关；首次三星给额外星 + 可能掉落装备/皮肤碎片。
- **早期保底爽感**：前 5 关默认给高概率强牌（分裂/穿透/燃烧），保证新玩家能看到尸潮被清屏，而不是只看到规则说明。
- **弱点不绑架**：普通关弱点是"更容易拿星"的推荐解；非推荐 build 仍可通关，只是更吃操作或更难满星。
