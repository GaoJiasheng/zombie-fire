# 10 · UI / UX

> 竖屏 9:16，逻辑分辨率 1080×1920。iOS 与 macOS **同一套 UI**，仅输入适配不同。

## 1. 屏幕流程图

```
启动 → [Splash] → [主菜单 main_menu]
  ├─ 开始 → [关卡之路 level_map] → 选关 → [出战配置 loadout] → [战斗 battle] → [结算 result] → 回 level_map
  ├─ 角色 → [角色/养成 character] （切角色、升技能初始等级、点分支）
  ├─ 装备 → [装备 equipment] （主炮/护甲/芯片/宝宝：装备、强化、解锁）
  ├─ 商店 → [商店 shop] （金币买装备 / 星解锁内容、皮肤）
  └─ 设置 → [设置 settings] （音量、语言、画质、存档、操作）
```

## 2. 主要界面清单与要素

| 界面 | ID | 关键要素 |
|---|---|---|
| 主菜单 | `screen_main_menu` | 标题Logo、开始、四大入口、当前角色立绘、金币/星显示 |
| 关卡之路 | `screen_level_map` | 竖向滚动的关卡链（类《消消乐》树），每关显示星数/锁、当前关高亮、章节分段 |
| 出战配置 | `screen_loadout` | 选角色、看属性、选主炮/护甲/芯片/宝宝、**选 1 自带通用技能**、本关弱点提示、开战按钮 |
| 战斗 | `screen_battle` | 见 §3 战斗 HUD，含目标锁定与目标策略 |
| 结算 | `screen_result` | 星级演出、金币/经验/星结算、首通/三星奖励、再战/继续 |
| 角色养成 | `screen_character` | 角色切换、等级/经验条、天赋点、技能列表与分支树（升初始等级、点分支） |
| 装备 | `screen_equipment` | 四槽位、装备详情、强化（金币）、解锁状态 |
| 商店 | `screen_shop` | 金币页（买装备）、星页（解锁角色/主炮/技能）、皮肤页 |
| 设置 | `screen_settings` | 音量、语言（简中，预留英文）、画质、操作说明、存档管理 |

## 3. 战斗 HUD（竖屏分区见 `01`）

- **顶部**：关卡号/波次进度条、暂停、基地血量条 + 护盾、当前关弱点小图标、当前目标策略图标。
- **战场**：怪物、子弹、特效、伤害数字（暴击放大变色）、金币掉落。
- **战场标记**：手动锁定圈、威胁标记、精英描边、免疫/抗性命中反馈（如"物理无效"）。
- **底部**：5 个技能槽（2 专属 + 3 通用，含 CD 转圈/可用高亮）、宝宝状态、局内经验条、reroll 次数。
- **三选一弹窗**：暂停式（或极慢速）弹出 3 张卡，点选；含"跳过/刷新/锁定"小按钮。卡面显示 tag、元素、是否顺当前 build。

## 4. 输入适配（同逻辑，双端）

| 行为 | iOS | macOS |
|---|---|---|
| 瞄准 | 触屏按住拖动 | 鼠标移动 / 按住左键拖动 |
| 放技能 | 点技能图标 | 点图标 或 数字键 1–5 |
| 切目标 | 双击僵尸 | 右键僵尸 |
| 切目标策略 | 长按锁定按钮 | 点击策略按钮 / Tab |
| 三选一选卡 | 点卡 | 点卡 / 数字键 1–3 |
| 菜单导航 | 点 | 点 / 键鼠（可选手柄后续） |
| 暂停 | 点暂停 | 点暂停 / Esc |

- 输入层抽象为统一的"指向(point) + 触发(trigger)"事件，业务逻辑两端共用（见 `13`）。
- 触控热区 ≥ 88×88px@1x（手指友好）；macOS 同尺寸但支持 hover 高亮/提示。
- 目标策略按钮用图标表达：准星=锁定、骷髅=精英/Boss、警报=越线威胁、心碎=血少。避免写长文字挤 HUD。

## 5. macOS 窗口缩放（等比放大，方案 A）

- 游戏以固定逻辑分辨率 1080×1920 渲染；窗口可自由拖拽放大缩小。
- **等比缩放（contain）**：画面整体放大/缩小填充窗口，多余区域用背景/留边（pillarbox），**游戏区域内容不变**——保证难度/视野与 iOS 完全一致、公平。
- 大屏 = 更大更清晰的同一份画面（不是看到更多战场）。
- Godot：`window/stretch/mode = canvas_items`，`aspect = keep`，根 `Camera2D`/CanvasLayer 固定逻辑尺寸。
- 记忆上次窗口尺寸/位置到存档（见 `13`）。
- 最小窗口尺寸限制（如 ≥ 405×720）避免 UI 过小。

## 6. 反馈与"爽感"（关键体验）

- **打击感**：命中闪白、轻微顿帧（仅暴击/击杀）、伤害数字弹跳、暴击数字更大更亮、击杀小爆。
- **屏幕震动**：仅大事件（Boss 登场、蓄能炮命中、自爆者爆炸），强度可在设置关闭。
- **金币吸入**：掉落金币向底部 HUD 飞入并计数跳动。
- **节奏高点提示**：精英出现/三选一弹出有明确音画提示（见 `12`）。
- **锁定反馈**：锁定目标有准星圈、轻微音效、炮口线性转向；切换目标策略时 HUD 图标短亮。
- **免疫/抗性反馈**：无效命中不要只飘 0，要明确显示"物理无效"或对应元素图标破碎，帮助玩家学习。
- **质变卡反馈**：Lv3 分支和 Lv5 终极卡要有更重的卡框、音效和进入战场后的可见变化。
- **无卡顿优先**：所有特效设上限与对象池（见 `13` 性能预算），宁可降特效不掉帧。

## 7. 可读性与可访问性

- 伤害数字、弱点图标、Boss 免疫元素用**颜色 + 形状双编码**（色盲友好）。
- 重要提示（"物理无效！""炮塔被冻结"）用文字 + 图标弹出。
- 字号、UI 缩放在设置可调（照顾大屏 macOS 与小屏 iPhone）。
- 暂停不影响理解：暂停界面可查看当前 build、怪物弱点。

## 8. 前期引导 UX

- 前 5 关不弹大段教程，尽量用按钮闪烁、目标圈、卡牌高亮和短提示完成教学。
- 第一次出现弱点/抗性时，选关界面和命中反馈各提示一次，之后收敛到图标。
- 第一次三选一固定给一张爽感牌、一张控制牌、一张经济/节奏牌，让玩家理解选择差异。
- 结算界面突出"刚刚变强了什么"：角色经验、金币强化可达、补星差额、下关推荐准备。
