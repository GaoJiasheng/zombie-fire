# 15 · App Production Plan

> 目标：把 Zombie Fire 从当前 M1 原型推进到可上架的 iOS/macOS App。
> 分工原则：核心代码与 Godot 实现可外包；产品方向、数据设计、素材生成、验收标准由我们控制。

## 1. 产品边界

### 首发 App

- 名称暂定：`Zombie Fire`
- 类型：竖版 Roguelike 塔防自动射击
- 引擎：Godot 4.x / GDScript
- 平台：iOS + macOS 首发
- 商业模式：无内购、无广告、无体力
- 存档：本地存档，无云同步
- 核心体验：固定底部炮塔自动开火，玩家手动瞄准/锁定，局内三选一升级，在 99 关中长期变强。

### v1 内容规模

| 模块 | v1 规模 |
|---|---:|
| 关卡 | 99 |
| 章节/场景 | 4 |
| 角色 | 4 |
| 主炮 | 8 |
| 护甲 | 6 |
| 芯片 | 8 |
| 宝宝 | 6 |
| 通用技能 | 16 |
| 专属技能 | 8 |
| 普通僵尸 | 20 |
| Boss | 8 |
| 元素 | 5 |

### 非目标

- 不做 IAP、广告、抽卡付费。
- 不做服务器、账号、云存档。
- 不做多人/PVP。
- 不做自由移动角色射击；炮塔固定是核心定位。
- 不在 M1/M2 期间加入 v1.1 新玩法。

## 2. 里程碑计划

| 里程碑 | 目标 | 主要产物 | 外包验收 |
|---|---|---|---|
| M1 | 可玩核心原型 | 5 关、1 角色、1 主炮、5 僵尸、1 Boss、4 技能、结算闭环 | macOS 可运行，3 分钟内看到选卡/清屏/结算 |
| M2 | 全量内容框架 | 全量数据表、99 关、4 角色、20 僵尸、8 Boss、全装备技能可用 | 全内容可进入、可解锁、无崩溃 |
| M3 | 手感与平衡 | 卡牌导演、数值曲线、反馈、音频、性能优化 | 60 FPS，前 20 关喂爽，99 关可过性达标 |
| M4 | 上架候选 | iOS/macOS 构建、商店素材、隐私配置、TestFlight | 真机可玩，朋友试玩通过 |
| M5 | 可选迁移 | Web/Windows/Android | 不新增玩法，仅迁移适配 |

## 4. 外包开发总原则

外包只负责实现，不负责重定义产品。

必须遵守：

- 所有内容走 `data/*.json`，代码读取数据。
- 素材路径与 ID 必须遵守 `design/data/naming_convention.md`。
- 不私自替换素材，不重新生成图，不改美术方向。
- 不扩大 v1 内容量。
- 每个里程碑都要保持可运行。
- 每个功能必须更新 `design/m1_todo.md` 或后续里程碑 todo。

## 5. 技术验收底线

每次交付至少跑：

```bash
python3 tools/validate_data.py
```

如果 Godot CLI 可用，还要跑项目启动检查。外包交付必须说明：

- 修改了哪些文件。
- 完成了哪些 todo。
- 未完成或有风险的点。
- 是否在 Godot 编辑器/真机上跑过。
- 当前 FPS 与主要性能风险。

## 6. M1 完成定义

M1 必须满足：

- 5 关可以从菜单进入、战斗、胜负、结算、返回。
- 炮塔自动开火。
- 鼠标/触控瞄准。
- 右键/双击锁定目标。
- 自动目标优先级可信。
- 局内 XP 触发三选一。
- 一局 1 次 reroll。
- 分裂、穿透、多重、减速能明显改变弹幕。
- Level 5 Boss 可击败，不出现物理免疫导致无解。
- 保存/读取进度可用。
- macOS 能跑，iOS 输入逻辑无工程分叉。

## 7. M2 完成定义

M2 必须满足：

- 99 关全部在 `levels.json` 中配置。
- 4 角色可选，属性与专属技可生效。
- 24 技能可出牌、可升级、可触发。
- 20 普通僵尸机制可区分。
- 8 Boss 机制可进入战斗。
- 主炮/护甲/芯片/宝宝可装备、强化、解锁。
- 关卡地图能显示星级与解锁进度。
- 所有内容使用 accepted 或 production-ready 素材。

## 8. M3 完成定义

M3 必须满足：

- 60 FPS 稳定，压力场景不明显掉帧。
- 对象池用于子弹、敌人、VFX、伤害数字。
- 卡牌导演不连续出烂牌，核心 build 成型率达标。
- 前 5 关极易上手，前 20 关持续给爽点。
- 99 关建议等级玩家至少可拿 2 星。
- 音效、命中、击杀、升级、结算反馈完整。
- 存档损坏可回退备份。

## 9. 上架准备

M4 需要准备：

- App icon。
- App Store 截图：iPhone、iPad、macOS。
- 预览视频：15-30 秒。
- 隐私说明：无账号、无广告、无内购、无数据收集。
- TestFlight 包。
- macOS notarization。
- 朋友试玩反馈表。

## 10. 决策日志

重要决策必须写入文档，不在聊天里消失：

- 改玩法：更新 `design/01_core_gameplay.md` 或相关玩法文档。
- 改数值：更新 `design/09_balance.md` 和数据表。
- 改技术：更新 `design/13_tech_architecture.md`。
- 改素材范围：更新 `design/assets/asset_manifest.md`。
- 改外包范围：更新 `design/16_outsourcing_brief.md`。
