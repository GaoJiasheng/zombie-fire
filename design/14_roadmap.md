# 14 · 路线图（Roadmap）

> 从"设计冻结"到"上架"，再到"多平台迁移"。每个里程碑有明确产物与验收。
> 原则：**先把核心循环做爽，再铺量**。原型阶段用占位素材，不等美术齐全。

## 里程碑总览

| 阶段 | 名称 | 目标 | 验收 |
|---|---|---|---|
| **M0** | 设计冻结 | 本 `design/` 目录定稿、确认 | 你 review 通过，开始写代码 |
| **M1** | 可玩原型 | 核心循环 + 第一分钟爽感跑通 | 能瞄准/自动开火/锁定/三选一/通关结算，清屏与命中反馈足够爽 |
| **M2** | 内容填充 | 全量内容 + 正式素材接入 | 99 关 / 4 角色 / 24 技能 / 20 僵尸 / 8 Boss 全可玩 |
| **M3** | 打磨平衡 | 数值、音效、UI、存档、两端适配 | 可过性矩阵达标、双端体验一致、无明显卡顿 |
| **M4** | 上架 | App Store（iOS+macOS） | TestFlight → 正式发布，朋友能玩 |
| **M5** | 迁移（可选） | Web / Windows / Android | 至少 Web 可玩 demo |

## M0 · 设计冻结（当前）
- 产物：`design/` 全部文档（已生成）。
- 待办：你 review，确认/微调 → 锁定，进 M1。

## M1 · 可玩原型（核心循环优先）
**目标**：用最少内容验证"好不好玩 + 顺不顺"。
- Godot 工程骨架（§13 目录）、autoload 单例、输入抽象、窗口缩放。
- 战斗场景：炮塔自动开火 + 拖动瞄准 + 1 把主炮（autocannon）。
- 目标系统：自动优先越线威胁，支持手动锁定与简单策略切换。
- 敌人系统（数据驱动）+ 5 个 T1 僵尸 + 波次出怪（读 levels.json）。
- 技能系统：3~4 个通用技能 + 局内经验条 + 加权三选一 + 1 次刷新 + run_level 升级。
- 基地血量 + 越线扣血 + 三星结算。
- 1 个 Boss（tank_titan）跑通免疫机制。
- 5 关（level_001~005）+ 简易选关 + 本地存档。
- 占位素材（色块/草图）即可；接 1~2 个真实音效验证手感。
- **验收**：iOS 真机 + macOS 各跑一遍，手感顺滑、无卡顿、循环闭环；玩家在 3 分钟内看到分裂/穿透清屏、精英奖励、选卡质变、结算变强。

## M2 · 内容填充
**目标**：把设计文档里的量全部做出来。
- 数据表全量填充：`characters/skills/weapons/armors/chips/pets/zombies/bosses/levels/economy.json`。
- 4 角色 + 8 专属技 + 16 通用技（含成长分支树）。
- 20 僵尸 + 8 Boss（各机制脚本）+ 精英词缀。
- 装备系统全量（主炮/护甲/芯片/宝宝 + 强化 + 解锁）。
- 99 关波次脚本。
- 正式素材按 `assets/` 批量产出并接入（先角色/怪/Boss，再 VFX/UI/背景/视频）。
- 全量音频接入。
- **验收**：全部内容可玩可解锁，无占位残留。

## M3 · 打磨与平衡
- 跑 `09 §6` 可过性矩阵 → 调表，确保达标玩家每关 ≥2 星、必可过。
- 跑 `09 §7` 卡牌导演模拟 → 调权重，确保核心 build 成型率与烂牌保底达标。
- 打击感/特效/音画反馈抛光（见 `10 §6`）。
- 双端一致性回归（输入、UI 缩放、性能）。
- 性能优化到稳定 60FPS（对象池、上限、画质档）。
- 存档健壮性（断电/异常退出/版本迁移）。
- 本地化文案集中到 `localization_zh.json`，校对。
- **验收**：可上架候选包，朋友试玩反馈通过。

## M4 · 上架
- App Store 准备：图标、截图、预览、隐私信息（无 IAP/无广告/无数据收集，填报简单）。
- iOS + macOS 双平台构建、签名、公证。
- TestFlight 内测 → 修缺陷 → 正式发布。
- 可选：Game Center 排行榜（仅炫耀，不影响平衡）。
- **验收**：朋友能在 App Store 下载并游玩。

## M5 · 多平台迁移（可选，架构已预留）
- **Web**：Godot HTML5 导出，处理包体/加载/`user://`(IndexedDB)，发布可玩 demo。
- **Windows**：导出 + 鼠标键盘（与 macOS 逻辑同源，工作量小）。
- **Android**：导出 + 多机型分辨率适配（触控与 iOS 同源）。
- 迁移期不新增玩法，只做平台适配与性能调优。

## 优先级取舍原则
1. **手感 > 内容量**：M1 必须先爽，否则铺量无意义。
2. **数据驱动 > 硬编码**：内容尽量进 JSON，方便后期调。
3. **占位先行**：不被美术/音频卡住进度，先用占位跑通逻辑。
4. **双端同源**：任何功能都同时验证 iOS + macOS，不留分叉。
5. **可过性是底线**：宁可简单点，不能出现"无解卡墙"。

## 风险与对策
| 风险 | 对策 |
|---|---|
| AI 出图风格漂移/分件一致性差 | 先定样张锁风格；分件法 + 人工切件兜底（见 `prompts_visual` §2/§10） |
| 数值难平衡 | 自检脚本 + 全局旋钮 `GLOBAL_*`，分章节灰度调 |
| 群战性能 | 对象池 + 同屏上限 + 画质档，宁降特效不掉帧 |
| Web 包体过大 | 素材分级加载、纹理压缩，Web 版可裁剪视频/高清档 |
| 范围蔓延 | 严守 v1 规模（§00），新点子记 backlog 留 v1.1 |
| 过度理性导致不够上头 | 前 20 关喂爽、质变卡、高反馈密度；M1 就验证第一分钟 |
| 可控随机变成假随机/无惊喜 | CardDirector 保留跨流派低概率惊喜牌，并提供有限刷新而非完全点菜 |
