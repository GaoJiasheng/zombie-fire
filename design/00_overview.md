# 00 · 项目总览

## 一句话定位

> **竖版 Roguelike 塔防射击**：你是末日中的炮台指挥官，自动开火、手动瞄准、局内选卡成型，在 99 关里越打越强，把一波波僵尸挡在基地之外。

对标《向僵尸开炮》，但在三件事上明确做得更好：
1. **数值更透明**——只有"攻/血"两条根属性，玩家算得清、我们调得动。
2. **技能更可控但仍有惊喜**——玩家自带核心技能，局内用加权三选一、刷新、锁定来驾驭随机。
3. **关卡更公平**——每关有建议等级与明确威胁，弱点是推荐解不是唯一解，靠"养成 + 操作 + 搭配"必定可过，不卡数值墙。

同时保留成熟同类产品真正有效的体验：**低操作门槛、尸潮压迫、一炮倒一片的反馈、局内成长的意外成神、长期变强的牵引**。本作去掉氪金压力后，必须用更密的手感反馈、更多可选挑战和更清楚的 build 目标补回长期动力。

## 目标平台

| 平台 | v1 | 说明 |
|---|---|---|
| iOS（iPhone/iPad） | ✅ 首发 | 纯点击交互 |
| macOS | ✅ 首发 | 鼠标交互，竖向窗口可自由缩放（等比放大） |
| Web (WASM) | 🔜 预留 | 引擎与架构已选可导出方案 |
| Windows | 🔜 预留 | 同上 |
| Android | 🔜 预留 | 同上 |

**核心约束：iOS 与 macOS 模型、数值、表现完全一致**，差异只在输入层（手指点击 vs 鼠标）。

## 技术选型（结论，理由见 `13_tech_architecture.md`）

- **引擎：Godot 4.x，语言 GDScript。**
  - 一套代码导出全部目标平台（含 Web，这是排除 Apple 原生 SpriteKit 的决定性原因）。
  - 2D/2.5D 表现顺滑、免费无版税、可上 App Store。
  - 选 GDScript 而非 C#：Godot 的 C# 对 **Web 导出**支持不完善，而我们要保留 Web 能力。
- **渲染：2.5D**——3D 质感的 2D 贴图（非真 3D 模型），用引擎骨骼动画做动效。
- **数据驱动**：关卡/技能/怪物/装备全走 JSON，便于平衡与"工具↔素材"连接。
- **存档**：本地文件持久化（关机重启不丢），无云、无跨设备（v1）。

## 设计支柱（一切取舍的准绳）

1. **看得懂的强大**——伤害数字飞出来要爽，且玩家能理解"为什么变强了"。
2. **有意图的 build**——出门前选好流派（元素/打法），局内围绕它加权随机，能歪出惊喜但不被烂牌绑架。
3. **公平的难度**——难，但可解。难度来自怪物机制、目标优先级和节奏，不是数值碾压或唯一元素答案。
4. **前期先爽再复杂**——前 20 关逐层开放系统，用明显质变技能和尸潮清屏建立信心，再逐步增加配装深度。
5. **零压力长线**——无体力、无氪金、无广告；用挑战关、无限尸潮、外观收集和全三星驱动长期游玩。
6. **两端同源**——iOS 与 macOS 是同一个游戏，不做阉割或差异化内容。

## 核心循环（详见 `01`）

```
选关 → 配置(角色/主炮/护甲/芯片/宝宝/带3~5技能) → 进入关卡
  → 自动开火 + 手动瞄准/锁定 + 局内加权选卡升级技能 → 击退所有波次/击杀Boss
  → 按基地剩余血量结算 1~3 星 + 掉落金币
→ 局外用金币强化装备 / 用经验升角色等级(得天赋点)升技能 / 用星解锁新内容
→ 下一关（难度↑，建议等级↑）
```

## 里程碑（详见 `14_roadmap.md`）

| 阶段 | 目标 | 关键产物 |

| M0 设计冻结 | 本 `design/` 目录定稿 | 全部设计文档 |

## 发布

- 上 App Store（iOS + macOS），**非盈利，给朋友玩**。
- 无内购、无广告。可选接入 Game Center 排行榜（仅炫耀，不影响平衡）。
- 上架后优先补"变异尸潮/无限尸潮/挑战词缀"这类轻内容，延长无氪单机寿命。
