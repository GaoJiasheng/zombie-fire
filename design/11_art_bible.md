# 11 · 美术风格圣经（Art Bible）

> 美术与 AI 出图的统一标准。**所有视觉素材都必须符合本文**，以保证 99 关、几百个素材风格一致。
> 逐素材的生成 prompt 见 `assets/prompts_visual.md`；清单见 `assets/asset_manifest.md`。

## 1. 总风格定调

- **2.5D 卡通写实（stylized semi-realistic, 3D-rendered look）**：像 3D 渲染出来的卡通图，但作为 2D 贴图使用。介于《向僵尸开炮》的清爽卡通与"略带末日质感"之间。
- 关键词：`stylized, semi-realistic, 3D-rendered 2D sprite, clean shapes, bold silhouette, soft global illumination, slight rim light, mobile game art, high readability`。
- **轮廓优先**：每个单位剪影一眼可辨（塔防里"看清是什么怪"高于细节）。
- **威胁优先**：自爆、复活、护盾、冲锋等高威胁机制必须在剪影和局内特效上一眼可读，不靠玩家点开说明。
- **明快但不廉价**：颜色饱和有活力，但有末日脏旧质感点缀（锈、血、灰），避免纯欢乐糖果风。
- **统一光源**：主光来自**画面右上方**，柔和环境光 + 单位边缘轻微 rim light，便于 2.5D 立体感。

## 2. 色板（Palette）

| 用途 | 颜色 | Hex（基准，可微调） |
|---|---|---|
| 主背景暗部 | 末日蓝灰 | `#1B2430` |
| 主背景亮部 | 雾霾灰 | `#3A4A5A` |
| UI 主色 | 警戒橙 | `#FF7A3D` |
| UI 强调/危险 | 血红 | `#E23B3B` |
| 金币/经济 | 金黄 | `#FFC83D` |
| 星/珍贵 | 星金 | `#FFE36B` |
| 物理元素 | 钢灰白 | `#D9DEE5` |
| 火元素 | 橙红 | `#FF5722` |
| 冰元素 | 冰蓝 | `#46C6FF` |
| 雷元素 | 电紫黄 | `#C77DFF` / `#FFE14D` |

- **元素色是硬约束**：火必橙红、冰必冰蓝、雷必紫黄、毒必毒绿、物理必钢灰白。子弹、技能图标、特效、对应主炮一律遵守，玩家靠颜色秒识别属性。

## 3. 比例与构图规范

- 设计基准分辨率 1080×1920；**出图统一按 @3x 等比的最高清一档**（见命名规范 §3），引擎降采样。
- **普通僵尸**：画布约 256×256（@1x），单位实际占比中等，群战不糊。
- **Boss**：画布约 768×768 或更大，体型为普通怪 2.5~4 倍，威压感强。
- **主炮/炮塔**：底部中央，约 320×320，细节可多（玩家长时间盯着）。
- **图标（技能/装备/货币）**：512×512 出图，方形，留透明边距，中心构图。
- **背景**：1080×1920 整图 + 可选分层（远景/中景/近景）做轻视差。

## 4. 骨骼分件规范（动效核心，呼应命名规范 §4）

会动的单位（角色/僵尸/Boss/宝宝）**不出整图**，而是出**透明背景分件 PNG**，引擎里骨骼绑定做动画：

- 标准分件：`head, body, arm_l, arm_r, hand_l, hand_r, leg_l, leg_r, weapon`（按造型增减）。
- 每件**独立透明背景、关节处留可重叠余量**（避免转动露缝）。
- 朝向：默认**面向上方/侧上**（怪向下走、面对炮塔）；提供一个**侧 3/4 视角**统一所有单位。
- 附一张 `{主名}_portrait.png` 合体参考图，供绑定时对位。
- 在引擎做的动作：`idle / walk / attack / hurt / death`（+ Boss/特殊怪的 `special`）。
- 复杂一次性表演（Boss 登场）可用**视频** `vid_boss_intro_*`（见 §6）。

> 这条决定了 `assets/prompts_visual.md` 里每个动态单位会给"分件出图"prompt，而非整图。

## 5. 特效（VFX）规范

- 技能/命中/爆炸特效：以**序列帧贴图**（sprite sheet）或**引擎粒子**实现，前缀 `vfx_`。
- 序列帧：透明背景、统一帧尺寸、帧数 8~16，命名 `vfx_explosion_fire_01..16.png`。
- 元素特效配色严格遵守 §2。
- 伤害数字为字体渲染（非贴图），暴击更大更亮（见 `10`）。
- 目标锁定与威胁标记属于战斗可读性资产：`vfx_target_lock` 清晰但不遮挡，`vfx_threat_warning` 只给高危敌人，避免全屏都是提示。
- 免疫/抗性反馈要有元素图标破碎或弹开感，让玩家立刻知道"不是没打中，是打不动"。

## 6. 视频素材规范（仅用于演出，非实时玩法）

- 用途：开场 CG、章节过场、Boss 登场。前缀 `vid_`。
- 竖屏 1080×1920，时长 3~8 秒，可循环或一次性。
- 风格须与 2.5D 基调一致（同色板、同光源）。
- 可由 AI 视频工具（如 Sora）生成；玩法中的单位动效一律不用视频（用骨骼/序列帧）。

## 7. UI 视觉规范

- 风格：科技军事 HUD + 末日做旧。圆角矩形面板、警戒橙描边、轻微发光。
- 按钮三态：normal / hover(macOS) / pressed，统一出图或九宫格拉伸。
- 字体 `font_main`：粗体无衬线，中文支持完整（含生僻词如"炮"），数字等宽便于跳动计数。
- 图标统一描边粗细与圆角半径。
- 三选一卡牌要区分普通升级、分支质变、终极进化、经济牌：用框色/角标/光效区分，不靠长文本。
- 目标策略图标必须小而清楚，适合底部/顶部 HUD 常驻。

## 8. 皮肤（外观）规范

- 皮肤改变角色/主炮/技能特效外观，**不改数值**（见 `02`）。
- 命名：在主名后加 `_skin_{name}`，如 `weapon_railgun_skin_gold.png`、`char_vanguard_skin_neon_portrait.png`。
- 同一对象的皮肤必须沿用同一骨骼分件结构（直接替换贴图即可换肤）。

## 9. 一致性检查清单（出图验收）

- [ ] 是否右上主光 + 边缘光？
- [ ] 元素配色是否符合 §2？
- [ ] 剪影是否一眼可辨？
- [ ] 动态单位是否为透明分件（非整图）？
- [ ] 尺寸/画布是否符合 §3？
- [ ] 命名是否符合 `data/naming_convention.md`？
- [ ] 高威胁敌人是否在 0.5 秒内能被识别？
- [ ] 锁定圈、威胁标记、免疫反馈是否清楚且不遮挡弹幕？
