# 02 · 数值模型与经济系统

## 1. 两条根属性：攻 / 血

沿用《向僵尸开炮》的极简设计——**全游戏只有两条根属性**，其余全部派生。这样玩家算得清、我们调得动。

| 根属性 | 含义 |
|---|---|
| **ATK 攻** | 输出能力的总和（炮塔每发子弹的基础伤害由它决定） |
| **HP 血** | 这里指**基地血量** `base_hp`（炮塔不会被直接打死，僵尸越线才扣血） |

> 注意：本作炮塔本体不承受贴脸伤害，"血"主要是**基地血量**这一道闸。这让"防御"的意义变成"别让怪越线"，与塔防定位一致。

### 派生属性（全部由攻/血 + 装备 + 技能修正得来）

| 派生属性 | 公式 / 含义 | 主要来源 |
|---|---|---|
| 暴击率 `crit_rate` | 触发暴击的概率 | 芯片、技能 `skill_critical`、角色 |
| 爆伤 `crit_dmg` | 暴击倍率（基础 ×1.5） | 芯片、技能 |
| 攻速 `fire_rate` | 每秒发数 | 主武器基础值 × (1+加成) × `PLAYER_FIRE_RATE_MULT`；当前全局节奏值为 `0.25` |
| 元素伤害 `elem_dmg_{x}` | 对应元素的增伤% | 芯片、技能、角色 |
| 减伤 `dmg_reduce` | 基地受到越线伤害的减免% | 护甲 |
| 穿透 `pierce` | 子弹可穿透的敌人数 | 主武器、技能 `skill_pierce` |
| 暴击外的特效率 | 分裂/弹射/追踪触发等 | 各对应技能 |

### 伤害结算公式（单发命中）

```
基础伤害   = ATK × weapon_dmg_coef × skill_mods × PLAYER_SHOT_DAMAGE_MULT
元素增伤   = × (1 + elem_dmg_{element})
弱点/抗性  = × weakness_mult     // 弱点1.5 / 普通1.0 / 抗性0.5 / 免疫0
暴击       = × (暴击命中 ? crit_dmg : 1)
最终伤害   = 上述连乘，取整
```

- DoT（火灼烧、毒中毒）：每跳按"快照时的最终伤害 × DoT系数"结算，持续 X 秒。
- 控制（冰减速/冻结、雷麻痹）：不直接造成大伤害，价值在节奏控制。

> 详细数值系数、随等级的成长曲线见 `09_balance.md`。`PLAYER_FIRE_RATE_MULT = 0.25` 负责当前射击手感，`PLAYER_SHOT_DAMAGE_MULT = 3.0` 负责单发补偿；本文只定义"框架与来源"。

## 2. 角色属性的三个来源

最终战斗属性 = **角色等级基础值** + **装备加成** + **天赋加成** + **局内技能修正**。

1. **角色等级**（局外，靠经验升）：决定 ATK / base_hp 的基础盘。不同角色基础成长曲线不同（见 `04`）。
2. **装备**（局外，靠金币强化）：主炮给 ATK 与弹道、护甲给 base_hp 与减伤、芯片给各派生、宝宝给额外输出/功能（见 `06`）。
3. **天赋**（局外，靠天赋点）：升技能初始等级、解锁技能成长分支、少量根属性加成（见 `03`/`05`）。
4. **局内技能**（局内，临时）：本关内对子弹/属性的临时增益，关束清零（见 `03`）。

## 3. 货币与资源（3 种长期 + 1 派生 + 1 局内）

| 资源 | 图标 ID | 获取 | 消耗 | 说明 |
|---|---|---|---|---|
| **金币 `gold`** | `icon_currency_gold` | 打怪掉落、首通奖励 | 强化/购买装备 | 主力消耗货币，可肝 |
| **经验 `xp`** | `icon_currency_xp` | 打怪、通关结算 | 自动→角色升级 | 不手动花，自动转化为等级 |
| **星 `star`** | `icon_currency_star` | 通关按剩余血量结算 | 解锁新主炮/技能/角色/皮肤 | 最珍贵，关联通关质量 |
| （派生）**天赋点 `talent_point`** | `icon_talent_point` | **每次角色升级自动获得** | 升技能初始等级、解锁成长分支 | 不单独刷，由升级派生 |
| （局内）**刷新点 `reroll_charge`** | `icon_reroll_charge` | 精英击杀、关卡节点、少量技能效果 | 三选一卡牌刷新/锁定 | 仅本局有效，不进入长期经济 |

> **为什么是 3 货币**：用户要求别复杂。经验不需玩家管理（自动升级），天赋点是升级的副产物。玩家真正"决策怎么花"的只有 **金币（养装备）** 和 **星（解锁内容）** 两条线，清爽。

### 经验 → 等级 → 天赋点链条

```
打怪/通关 → 攒 xp → 角色升级（xp 阈值递增）
        → 每级 +角色基础属性
        → 每级 +N 个 talent_point（前期1点/级，后期2点/级，曲线见 09）
talent_point → 升技能初始等级 + 解锁技能成长分支（见 03/05）
```

### 金币经济

- **产出**：每只僵尸有 `gold_drop` 区间；精英/Boss 更高；首通额外大笔。失败也保留本局已掉金币。
- **消耗**：装备强化（主炮/护甲/芯片/宝宝），消耗随等级指数上升。
- **平衡目标**：玩家在某关卡停留刷 2~4 遍，攒的金币足够把核心装备升到"建议等级对应档"，从而推进（曲线见 `09`）。
- **无金币墙**：不存在"必须氪/必须刷到天荒地老"的卡点；任何关卡的过关所需养成度都设计为"可在合理重刷次数内达到"。

### 星经济

- **产出**：99 关 × 3 星 = 297 星上限。
- **消耗**：解锁内容定价（示例，详见 `09`）：
  - 解锁第 2/3/4 个角色：例如 30 / 50 / 70 星。
  - 解锁高级主炮（如 `weapon_plasmacannon`）：例如 18 星。
  - 解锁高级通用技能进池：每个 5~15 星。
  - 皮肤（纯外观）：剩余星随意花。
- 总解锁成本控制在 **≈220 星**以内，留富余，保证正常通关玩家能解锁全部玩法内容（皮肤为可选消费）。
- 星不直接买战斗数值；它买的是**玩法宽度**（角色、武器、技能进池）和外观。这样三星有价值，但不会变成"不满星就卡战力"。

## 4. 装备强化经济（概览，细节见 `06`/`09`）

- 每件装备有 `level`（强化等级），强化消耗 `gold`，效果线性或分段提升。
- 强化只升级**已拥有**的装备；获得新装备靠星解锁或关卡掉落。
- 不做"洗练/随机词条/抽卡"——保持透明可预期（与"看得懂的强大"支柱一致）。
- 不做付费式材料黑洞。强化曲线允许玩家短期刷出结果，长期目标放在多 build、多角色、挑战词缀与外观收集。

## 5. 无氪长线目标

去掉内购和体力后，长期动机不能靠压力制造，必须靠"我还想试一局"：

| 系统 | 奖励 | 战力影响 |
|---|---|---|
| 全三星回刷 | 星、皮肤、章节徽章 | 间接解锁玩法宽度 |
| 章节挑战关 | 金币、外观碎片、排行榜成绩 | 不给唯一战力 |
| 变异尸潮 | 当日/当周特殊词缀，给金币和皮肤碎片 | 可刷但不必刷 |
| 无限尸潮 | 纯挑战与 build 验证 | 不影响主线平衡 |
| 图鉴收集 | 怪物/Boss/技能记录 | 纯展示 |
